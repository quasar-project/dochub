◆ unreachable

#define unreachable ( )     throw std::runtime_error("unreachable code reached");

Помечает код как недостижимый

Отмечает код как недостижимый для компилятора. Необходимо для предотвращения предупреждений компилятора/анализаторов о том, что не все пути выполнения функции возвращают значение.

Пример использования:

auto loop() -> int
{
while(true)
{
if(cond)
return 0;
}
$unreachable()
}

Если код, отмеченный таким образом, все же будет выполнен, то будет вызвано исключение.

Исключения
std::runtime_error
Уст.
Макросы - это плохо. Мы больше не используем макросы. Вместо этого макроса теперь используется встроенная стандартная функция std::unreachable (с С++23 и выше), либо ее полный аналог leaf::unreachable.
См. также
leaf::unreachable

◆ unimplemented

#define unimplemented ( )     throw std::runtime_error("unimplemented code");

Помечает участок кода как не имеющий реализации

Макрос удобен для замены комментариев с todo. Если код, отмеченный таким образом, все же будет выполнен, то будет вызвано исключение.

Исключения
std::runtime_error
Уст.
Макросы - это плохо. Мы больше не используем макросы. Вместо этого макроса теперь необходимо явно написать выброс исключения, или любой другой способ завершения программы/вызова ошибки.

◆ trace

#define trace ( ...)    SPDLOG_TRACE(__VA_ARGS__)

Выводит форматированное сообщение с уровнем логгирования Trace.

Макрос принимает литерал форматирования и произвольное число аргументов для форматированного вывода. Аргументы форматирования должны иметь спецификацию fmt::formatter. Если в активном логгере стоит уровень выше, чем Trace, то содержимое макроса не будет выведено. Пример:

$trace("Hello, {}!", "world");

Вывод:

Hello, world!

https://github.com/gabime/spdlog

Уст.
Макросы - это плохо. Мы больше не используем макросы. Вместо этого макроса теперь используется функция simkernel::log::trace, а для добавления имени функции - simkernel::source_location.
См. также
simkernel::log::trace
